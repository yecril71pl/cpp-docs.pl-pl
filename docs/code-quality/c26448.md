# <a name="c26448-use_gsl_finally"></a>C26448 USE_GSL_FINALLY

Rozważ użycie `gsl::finally`, jeśli ostateczna akcja jest zamierzona.

C++Podstawowe wytyczne: [GSL. util: narzędzia](https://github.com/isocpp/CppCoreGuidelines/blob/master/CppCoreGuidelines.md#SS-utilities)

Biblioteka wsparcia dla wytycznych zapewnia wygodne narzędzie do wdrożenia *ostatecznej koncepcji działania* . Ponieważ C++ język nie obsługuje konstrukcji **try-finally** , stała się powszechna, aby zaimplementować niestandardowe typy oczyszczania, które spowodują wywołanie dowolnego działania w przypadku niszczenia. Narzędzie `gsl::finally` jest implementowane w ten sposób i zapewnia bardziej jednolity sposób wykonywania ostatecznych działań w bazie kodu.

Istnieją również przypadki, w których ostateczne działania są wykonywane w starym, w stylu języka C, za pomocą instrukcji **goto** (co jest ogólnie odradzane przez [C26438 NO_GOTO](c26438.md)). Trudno jest wykryć dokładny zamiar w kodzie, który intensywnie używa instrukcji **goto**, ale niektóre heurystyke mogą pomóc w znalezieniu lepszych kandydatów do czyszczenia.

## <a name="remarks"></a>Uwagi

- Ta reguła jest bardzo lekka i używa nazw etykiet do odgadnięcia o szansach sprzedaży do korzystania z obiektów akcji finalnej.
- Nazwy etykiet, które mogą zgłosić ostrzeżenie, zawierają słowa takie jak "End", "Final", "Clean" i tak dalej.
- Ostrzeżenia są wyświetlane w instrukcjach **goto** . W niektórych przypadkach może być widocznych pełnych danych wyjściowych, ale może to pomóc w ustalaniu priorytetów kodu w zależności od jego złożoności.
- Ta reguła zawsze przechodzi do pary z [C26438 NO_GOTO](c26438.md). W zależności od priorytetów można wyłączyć jedną z nich.

## <a name="example-cleanup-with-multiple-goto-statements"></a>Przykład: czyszczenie przy użyciu wielu instrukcji goto

```cpp
void poll(connection_info info)
{
    connection c = {};
    if (!c.open(info))
        return;

    while (c.wait())
    {
        connection::header h{};
        connection::signature s{};
        if (!c.read_header(h))
            goto end;               // C26448 and C26438
        if (!c.read_signature(s))
            goto end;               // C26448 and C26438
        // ...
    }

end:
    c.close();
}
```

## <a name="example-cleanup-with-multiple-goto-statements-replaced-by-gslfinally"></a>Przykład: czyszczenie z wieloma instrukcjami goto zamienione przez GSL:: finally

```cpp
void poll(connection_info info)
{
    connection c = {};
    if (!c.open(info))
        return;

    auto end = gsl::finally([&c] { c.close(); });
    while (c.wait())
    {
        connection::header h{};
        connection::signature s{};
        if (!c.read_header(h))
            return;
        if (!c.read_signature(s))
            return;
        // ...
    }
}
```
